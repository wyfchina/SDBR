# S-DBR 新订单承诺评估与 CCR 预留设计

## 1. 目标与边界

本设计把真实新订单的接单判断从手工增量分钟试算升级为自动影子排程。系统根据订单的可执行路线、CCR 负荷、资源日历、已有排程、容量预留和可选物料检查，给计划员提供接单建议；计划员保留最终决定权。

本能力属于 DDOM / S-DBR 执行层，不治理 DDS&OP 主设置，不修改 DDAE 批准参数，不自动接受客户订单，不自动修改正式排程，也不执行 ERP/WMS 库存扣减。

现有 `BE-SDBR-005` 手工 What-if 继续用于询价、临时假设和不完整订单试算。真实订单必须进入独立的“订单承诺评估”工作台和自动影子排程流程。

页面文案使用“冲击会不会击穿约束”。其业务定义是：在冻结计划、有效日历、已接受容量预留和现有市场承诺不被破坏的前提下，找不到满足请求交期的 CCR 可用时隙。

## 2. 已确认业务原则

1. SDBR 只返回建议，计划员决定是否接受订单。
2. 即使订单能够按期完成，只要 CCR 负荷超过批准的保护线，也必须由计划员确认。
3. 计划员接受订单后，系统立即建立 CCR 容量预留；正式 Planning Run 随后把预留转为正式工序计划。
4. CCR 采用日期/班次级负荷分钟预留，不锁定精确工序开始和结束时间。
5. 正式排程失败时，预留保持有效并进入“排程异常待处理”，系统不得自动释放。
6. 物料可用性检查默认启用。计划员可以关闭，但必须填写原因。
7. 关闭物料检查时，系统只做 CCR 容量和交期判断，不能声称生产完全可行。
8. 物料检查启用且通过时，接受订单会同时创建物料计划分配；关闭检查时只登记待确认需求。
9. 接单阶段跳过物料检查不能绕过工单释放阶段的物料硬门控。
10. CCR 保护线来自 DDAE 批准并冻结的运行模型配置。若契约未提供该值，SDBR 只能用 80% 做明确标记的参考试算，不能给出普通“建议接受”。

## 3. 输入与权威来源

### 3.1 新订单输入

真实订单至少需要：

- 订单 ID、产品 ID、数量、请求交期、业务优先级；
- 可执行 Routing、工序顺序、标准工时、可选资源；
- 订单来源、接收时间和可追溯 ID。

订单可通过当前 Mock API 进入；未来 Direct ERP Adapter 或 UNS 接入只能替换连接方式，不能改变本设计的 canonical 业务语义。

### 3.2 SDBR 执行输入

- 当前正式排程和已锁定工序；
- 已接受但尚未转为正式排程的 CCR 容量预留；
- SDBR 资源日历、班次和有效产能窗口；
- 最新且有效的库存、已分配库存、在途和质量状态快照；
- SDBR 内已有的物料计划分配。

### 3.3 DDAE 冻结配置

- Operating Model Configuration 及其 fingerprint；
- Scheduling Configuration；
- CCR / 控制点和资源角色；
- CCR 保护线；
- 时间缓冲、冻结窗口、协商窗口和释放策略。

SDBR 只消费这些值。若现有契约缺少 CCR 保护线，不得新增隐式字段；必须向 Contract Agent 提交契约变更请求。

## 4. 自动影子排程

自动影子排程不修改正式排程，其步骤为：

1. 校验订单、路线、工时、资源、日历和配置引用。
2. 识别订单经过的 CCR；订单多次访问 CCR 时保留多段负荷。
3. 根据数量、标准工时、资源效率和已批准的换型规则计算 CCR 有效负荷分钟。
4. 读取正式 CCR 负荷和未转正容量预留，形成当前可承诺能力视图。
5. 从请求交期向前扣除已批准的下游保护时间，得到最晚 CCR 完成时间。
6. 在有效日期/班次窗口中寻找可容纳新增负荷的候选窗口。
7. 计算每个候选窗口的负荷前后值、保护线状态和最早安全承诺日期。
8. 按默认启用状态检查物料可用性；关闭检查时记录跳过原因。
9. 生成建议，不创建正式工序，不修改冻结计划。

Planned Load 百分比只用于风险判断，不能单独决定交期。某个班次负荷较低但剩余时隙晚于订单要求时，订单仍不可按期；某个班次负荷已满但后续班次仍能满足交期时，可以给出后续安全日期。

## 5. 建议结果

| 结果 | 判定 | 计划员可执行动作 |
|---|---|---|
| 建议接受 | 物料通过、交期可实现、CCR 未超过批准保护线 | 接受、调整交期、拒绝 |
| 需计划员确认 | 交期可实现，但 CCR 超过批准保护线 | 确认接受、调整交期、拒绝 |
| 产能可接受，物料待确认 | 关闭物料检查，CCR 与交期可行 | 有条件接受、重新启用物料检查、拒绝 |
| 待物料确认 | 启用物料检查但证据不足 | 刷新物料证据、调整交期、拒绝 |
| 建议调整交期 | 请求日期不可实现，但存在后续安全日期 | 接受建议日期、人工协调、拒绝 |
| 暂不建议接受 | 路线、工时、资源、日历、保护线或关键引用不足 | 补充数据、重新评估、拒绝 |

使用 80%参考保护线时，结果必须显示“默认参考，需计划员确认”，不得返回普通“建议接受”。

## 6. 计划员确认与预留

计划员接受订单时，系统在同一事务中：

1. 重新校验评估依据是否仍为最新版本；
2. 检查对应日期/班次容量是否仍可预留；
3. 创建 CCR 容量预留；
4. 物料检查通过时创建物料计划分配；
5. 物料检查被跳过时创建待确认物料需求，不创建已分配声明；
6. 将订单标记为“已接受、待正式排程”；
7. 记录计划员、确认时间、评估版本和审计原因。

CCR 预留至少包含：订单、产品、CCR、日期、班次、预留分钟、最晚 CCR 完成时间、配置版本、日历版本、评估 ID、计划员和状态。多 CCR 或重复访问 CCR 时生成多条预留明细。

预留不固定班次内的精确顺序。正式 Planning Run 可以在班次和协商窗口内调整，但不得突破最晚 CCR 完成时间；跨班次或跨日期必须重新评估承诺。

## 7. 正式排程衔接与异常

正式 Planning Run 必须消费待转正容量预留和物料计划分配：

- 成功：预留转为正式 CCR 工序占用，避免重复计入 Planned Load；
- 失败：预留和物料计划分配保持有效，状态为“排程异常待处理”；
- 取消：只有计划员明确取消订单或解除承诺后，才释放预留和计划分配；
- 变更：数量、交期、路线、CCR 或保护线变化时，原评估失效并要求重新评估。

系统不得因 worker 失败、超时、服务重启或排程不可行而静默释放已接受承诺。

## 8. 组件边界

- **Order Intake Adapter**：接收 canonical 新订单；连接方式可替换。
- **Order Commitment Evaluation**：编排引用校验、物料检查和影子排程。
- **CCR Shadow Scheduler**：寻找日期/班次候选窗口，不修改正式排程。
- **Material Feasibility Gate**：默认检查可用性；支持有审计的计划员跳过。
- **Recommendation Engine**：生成业务建议、风险和最早安全日期。
- **CCR Reservation Ledger**：防止并发接单重复承诺同一能力。
- **Material Planning Allocation Ledger**：记录计划分配或待确认需求，不修改 ERP/WMS 库存账。
- **Planning Run Bridge**：将已接受预留转为正式排程输入并回写转正或异常状态。
- **订单承诺评估工作台**：集中展示待评估订单、建议、证据、确认动作和预留状态。

现有排程结果 What-if 保持独立，只做人工假设试算，不直接创建订单承诺或容量预留。

## 9. UI 设计边界

新增独立“订单承诺评估”工作台，不放入创建排程向导，也不替代排程结果 What-if。

工作台默认显示：订单、请求交期、最早安全日期、CCR、负荷前后、保护线来源、物料状态、建议结果、预留状态和异常状态。订单详情显示路线负荷依据、日期/班次候选、物料证据和审计信息。

“检查物料计划可用性”默认开启。关闭时要求填写原因，并持续显示“物料待确认”。技术 ID、fingerprint 和原始原因码放在可展开详情中，不作为业务主信息。

## 10. 并发、幂等与审计

- 同一订单版本的自动评估必须幂等；重复消息不生成重复评估。
- 接受动作必须使用评估版本或乐观锁；基础负荷变化后拒绝过期确认并要求重新评估。
- CCR 预留与物料计划分配必须原子写入，不能只成功一半。
- 取消、调整、重评和转正均保留事件记录，不覆盖历史证据。
- 每次判断保留订单版本、配置版本、日历版本、运行快照、数据来源、时间戳和 trace ID。

## 11. 验收测试范围

1. 保护线以内且物料通过时返回“建议接受”，但不自动接受订单。
2. 超过保护线但仍按期时返回“需计划员确认”。
3. 无法在最晚 CCR 完成时间前找到窗口时返回建议安全日期。
4. Planned Load 不能脱离时间窗口单独决定交期。
5. 关闭物料检查时返回“产能可接受，物料待确认”，并记录原因。
6. 物料证据不足时不得返回普通“建议接受”。
7. 接受订单后创建日期/班次级 CCR 预留，防止并发重复承诺。
8. 物料通过时建立计划分配；跳过时只登记待确认需求。
9. 正式排程成功后预留转正且不重复计负荷。
10. 正式排程失败后预留保持锁定并产生异常。
11. 缺少 DDAE 正式保护线时只能使用标记清楚的 80%参考试算。
12. 订单、配置、日历或快照变化后，旧评估不能直接确认。
13. 工单释放阶段仍执行物料硬门控，不受接单阶段跳过影响。

## 12. 非目标

- 不自动接受客户订单；
- 不自动调整客户承诺日期；
- 不自动重排正式计划；
- 不在 SDBR 配置或审批 CCR 保护线和其他 DDAE 主设置；
- 不修改 ERP/WMS 库存账，不执行发料或移库；
- 不把影子排程称为正式 Planning Run；
- 不用 Simio 替代日常快速接单判断；复杂多资源耦合场景可另行建议 Simio 复核。

## 13. 后续规格映射

实施前应在 `docs/backend-specification.md` 为自动订单承诺评估、CCR 容量预留、物料计划分配和 Planning Run 转正新增或扩展 `BE-SDBR-*` / `BE-RUN-*` 验收项；在 `docs/ui-specification.md` 新增独立订单承诺评估工作台的 `UI-*` 验收单元。

若 DDAE 契约当前没有 CCR 保护线字段，应先通过 Contract Agent 提交契约变更请求，不能在 SDBR 中创建隐式替代字段。
