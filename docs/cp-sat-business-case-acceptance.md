# CP-SAT 业务案例验收手册

本文档用于人工判断 OR-Tools CP-SAT 排程结果是否符合业务直觉。案例全部使用 `TST-CP-*` 前缀，只属于测试系统，不代表生产数据。

相关能力：`BE-DATA-014`、`BE-SOLVER-009`、`BE-SOLVER-010`、`BE-SOLVER-011`、`BE-SOLVER-014`、`UI-OVERVIEW-001`。

## 使用方法

1. 重建或启动测试库。
2. 执行 `TST-CP-RUN-*` Planning Run。
3. 打开 `GET /planner/workbench/test-data/acceptance` 或计划总览中的“案例验收总览”。
4. 查看 `CP-SAT业务案例` 分组。
5. 重点看三列：期望断言、通过断言、差异原因。

验收原则：业务案例验证的是行为，不要求 CP-SAT 在多个等价最优解中总是选择完全相同的工序顺序。

## 案例清单

| 案例ID | 业务主题 | 期望结果 |
| --- | --- | --- |
| `TST-CP-FINITE-RESOURCE` | 约束资源有限产能 | 同一约束资源上的工序不重叠，全部订单都有排程条。 |
| `TST-CP-ALTERNATE-RESOURCE` | 备用资源选择 | 主资源拥堵时至少一项工序使用备用资源。 |
| `TST-CP-CALENDAR-OVERTIME` | 日历维护与加班 | 维护窗口不排产，加班窗口可以被使用，覆盖记录被冻结并应用。 |
| `TST-CP-RESOURCE-EFFICIENCY` | 资源效率修正 | 60 分钟标准工序在 50% 效率资源上占用 120 分钟。 |
| `TST-CP-SETUP-SEQUENCE` | 顺序相关换型 | 不同产品族之间保留 45 分钟换型间隔。 |
| `TST-CP-INFEASIBLE-WINDOW` | 不可行诊断 | 时间窗过窄时进入 `DeadLetter`，求解器诊断包含 `ORTOOLS_INFEASIBLE`。 |

## 业务解释

### 有限产能

这个案例回答“瓶颈设备是否会被排出同一时段加工两张工单”。通过条件是同一资源上所有加工条前后衔接或留有空档，没有时间重叠。

### 备用资源

这个案例回答“系统是否会在主资源拥堵时合理使用备用资源”。通过条件不是固定某张工单必须去备用资源，而是最终计划中确实出现备用资源使用。

### 日历覆盖

这个案例回答“临时维护和加班是否真的进入 CP-SAT”。基础班次为 08:00-12:00，维护覆盖扣掉该窗口，晚间 18:00-20:00 加班补充产能。通过条件是加工不落入维护窗口，并使用加班窗口完成。

### 资源效率

这个案例回答“设备效率是否改变工序占用时间”。标准工时 60 分钟，资源效率 50%，实际排程占用应为 120 分钟。

### 换型

这个案例回答“产品族切换是否产生换型间隔”。A 与 B 双向切换均为 45 分钟，因此不管 CP-SAT 选择哪个顺序，两项工序之间都应至少间隔 45 分钟。

### 不可行诊断

这个案例回答“排不出来时系统是否明确告诉原因”。一项 120 分钟工序被限制在 60 分钟时间窗内完成，正确结果不是勉强排产，而是 `DeadLetter` 加 `ORTOOLS_INFEASIBLE`。

## 当前边界

- 不验证 BOM/MRP、多级物料、替代料。
- 不验证批次、合批、拆批。
- 不验证多并行资源下的机台级换型。
- 不验证 ERP/MES/Simio 真实接口。
- 不把某个等价最优排序写成唯一正确答案。
