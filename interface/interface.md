### 2. 缓冲/优先级执行看板 (Buffer Board)

这是视频中重点演示的现场执行核心界面，用于监控工单流向瓶颈工序（如 C1 约束资源）的实时健康度。

* **顶部控制区：** 可通过下拉菜单筛选 `Location`（工厂/车间）与 `Resource`（特定的瓶颈资源，如 C1），并显示当前资源的当日总负荷小时数。
* **状态看板矩阵：**
* 纵向分为两组：**Yet to be Received（待接收工序）** 与 **Received（已接收并正在加工）**。
* 横向根据“时间缓冲”的消耗程度划分为五大状态列：`Early`（过早，浅蓝）、`Green`（安全，绿）、`Yellow`（注意，黄）、`Red`（高危，红）、`Late`（迟到，深红）。


* **悬浮详情弹窗（异常处理与详情）：** 鼠标悬浮在任意工单（如 101001-5）上时，系统会自动弹出浮窗，显示该工单的 **Part ID**（物料号）、**Customer**（客户名称，例如 Starbucks）、**Quantity**（加工数量）、**Promise Date**（承诺交期）以及 **Priority**（执行优先级编号），便于调度员在不切换页面的情况下直接审查现场异常。

### 5. 排程创建与核心参数设置页面 (Scheduling Settings)

视频中，演讲者通过导航进入了系统的核心控制台，其底层的参数控制分为两个核心配置页面：

#### 5.1 全局排程算法配置页 (`Processing Tab -> Scheduling Settings`)

该页面用于定义排程引擎如何解析物理世界的生产速率与时间：

* **产能速率解析模式（Run Rate Interpretation）：** 系统支持三种标准切换，用于适配不同的行业主数据：
* *Units per hour*（件/小时，适用于高频重复制造）
* *Hours per unit*（小时/件，适用于大型重工业定制制造）
* *Minutes per unit*（分钟/件，适用于精细离散制造）


* **缓冲时间单位（Buffer Times）：** 可选配置为 `Hours` 或 `Minutes`。
* **发货偏好时间（Preferred Shipping Time）：** 用于反向倒推完工发货缓冲（Completion Buffer）的锚点。
* **事务捕获配置（Transaction Settings）：** 包含两个复选框——`Record Early Arrival`（记录过早到达）与 `Transact Late`（事务逾期记录），系统通过这两个开关来决定是否在车间流水中捕获偏离计划的微观变动，用于后续的敏捷性基准（Baseline）分析。

#### 5.2 现场执行优先级激活页 (`Site Admin -> Execution Priority`)

系统允许用户跳过传统的“单纯依赖交期（Due Date）”的排程逻辑，一键启用**需求驱动执行优先级矩阵**：

* **优先级 1（最高）：** 对应当前已经发生缺货（Stocked Out）的备库缓冲物料（Buffered Parts）。
* **非缓冲件逾期逻辑（Non-buffered Parts Rule）：** 如果某张非备库工单的“理想开工日期（Ideal Start Date）”因为产能排挤已经落入过去（Past Due），系统会自动提升其执行优先级，并在排程列表中动态前置。

### 6. 主数据管理与资源编辑器 (Master Data & Resource Editor)

在系统的基础建模空间内，计划员需要对车间的资源模型（Resource Models）和时间进行抽象：

#### 6.1 弹性工作日历模型 (Time Schedules)

为了完全拟合真实的工厂环境，系统主数据支持三层嵌套定义：

1. **日定义 (Day Definitions)：** 设定单一工作日的有效工时。
2. **周定义 (Week Definitions)：** 设定标准的周工作班次（如单班、双班、五天或七天制）。
3. **班次覆盖与排除 (Shift Overrides & Exclusions)：** 允许计划员在日历上直接拉取特定时间段，强制注入“设备计划维保（Maintenance Periods）”或“临时停机排除”。

#### 6.2 资源分类编辑器 (Resource Editor)

资源（设备/产线）在系统中被强制划分为三大战略属性，其排程逻辑完全不同：

* **Constraint（约束资源 / 鼓）：** 开启有限产能排程。系统根据日历（如单班 8 小时）进行刚性截顶，多余的工单自动向后平移，绝对不产生产能虚高。
* **Buffered Resource（带缓冲的非约束资源）：** 采用无限产能排程，但允许在工艺路线上设置前置时间缓冲（Time Buffer）。典型应用场景如：外协电镀工序或高变动性的热处理工序。
* **Resource（普通非约束资源）：** 采用无限产能排程。系统默认其拥有充足的“冲刺产能（Sprint Capacity）”来吸收瓶颈工序传导过来的波动。

### 7. 排程作业列表与多视图控制 (Scheduler Workspace)

这是主生产计划员日常驻留的桌面（Workspace），其排程结果由三大元素协同呈现：

#### 7.1 作业序列网格 (Job List Grid)

排程创建后，所有生成的工单默认以 `Planned Release Date`（计划释放日期）在网格中进行升序排列。每一行工单包含以下关键核对字段：

* `End Request Date`（客户最晚需求日期）
* `Promised Date`（系统产能验证后的承诺交期）
* `Status Badge`（状态标签：高亮显示 **On-Time**、**Late** 或 **Early**）

#### 7.2 动态工艺路线与备用切换弹窗 (Routing & Alternate Routing Popup)

在网格中点击任意工单（Work Order），页面底部会立即展开该工单的完整多道工序流（如：下料 -> 粗加工 -> C1约束工序 -> 表面处理）。

* **动态缓冲显性化：** 系统在 C1 约束工序的前方，会显性化绘制一个 **2 Hours Time Buffer**（2小时时间缓冲区），提醒计划员该工单在到达瓶颈前有2小时的容错垫片。
* **备用路线按钮（Alternate Routings）：** 如果计划员发现某条产线极度拥堵，可点击此按钮弹出工艺路线库，直接一键将该工单分流至备用资产，系统会自动重新计算该工单在全厂的排程节点。

### 8. 现场异常处理、原因码弹窗与车间报工流

视频后半段演示了车间调度主管或操作工在**缓冲执行看板 (Buffer Board)** 上的真实闭环交互，这属于典型的 MES/Shop-floor 报工流：

```
[工单在车间流转] -> [落入Buffer Board红色Late区] -> [操作工点击工单激活Transact浮窗]
                                                               ↓
[系统强制要求选择缺陷原因码(Reason Code)] <- [触发"Received at Buffer"事务]
         ↓
(例如选择: Material Shortage 缺料)
         ↓
[点击"Start Quantity at Drum"开始加工] -> [工单正式从Buffer Board中销账/隐藏]

```

* **操作流触发：** 当车间发现某张工单（如由于上游故障或物料未就位）未能准时到达瓶颈工序的前置缓冲区时，该工单会在 Buffer Board 上移动到最右侧的 **Late（深红）** 区域。
* **异常拦截弹窗：** 操作工必须点击该工单，系统会弹出一个事务处理浮窗。操作工执行 `Received at Buffer`（在缓冲区接收）时，**系统会强制弹出一个下拉菜单要求选择“延迟原因码（Reason Code）”**。视频中演示了选择 `Material Shortage`（材料短缺）这一经典工业供应链痛点。
* **数据闭环与销账：** 选择完原因码并录入实际接收数量后，操作工点击 `Start Quantity at Drum`（在约束设备上开工），该工单的数据状态随即转入“进行中”，并自动从 Buffer Board 监控面板上隐藏。

### 9. 数据集成、智能分析与高级可视化视图 (Power BI & Analytics)

系统的高级分析层采用了与 Power BI 深度的嵌入式集成，为管理层提供了两张核心看板：

* **作业调度甘特图视图 (Gantt Charts)：** 纵轴为车间的物理设备/资源代码，横轴为时间线。每条工单表现为横向色块，最独特的是，**时间缓冲（Time Buffers）在甘特图上呈现为透明或特定条纹的占位块**。这能够让计划员一眼看出哪些设备之间留有了足够的“安全垫片”，哪些部分的排程过于紧凑、极其脆弱（Brittle）。
* **实际工艺路线拓扑流（Process Mining / 流程挖掘）：** 系统抽取车间历史实际报工的流转时间戳，逆向生成一张物流拓扑网络图。
* **流量热力显示：** 视频中线条的粗细代表了工单流转的物理频次和容量（Volume）。
* **重定向环路暴露 (Redirect Loops)：** 该视图能敏锐地抓取到某些工单在特定设备（如 `r10` 资源）上进行了多次返工或二次加工的循环线条。这有助于 S&OP 团队在战略层面上诊断车间的“隐形瓶颈”，从而修改主数据中的标称前置时间（Lead Time）。
